# PLB.004 Knowledge Object Assessment

## Purpose

Provide a repeatable process for evaluating whether a candidate knowledge item should be added to the Knowledge Repository and how it should be represented.

## Process

### Step 1 – Determine Knowledge Object Type

Classify the candidate as one of:

- Capability
- Opportunity
- Learning
- Constraint
- ADR
- Playbook
- Roadmap

### Step 2 – Assess Repository Worthiness

Evaluate whether the knowledge is:

- durable
- reusable
- broadly applicable
- valuable beyond the current session

### Step 3 – Perform Duplicate Review

Determine whether the knowledge already exists:

- directly
- partially
- under a different object type

### Step 4 – Perform Scope Overlap Analysis

Classify overlap as:

- Unique
- Partial Overlap
- Strong Overlap
- Duplicate

### Step 5 – Review Relationships

Identify:

- existing relationships
- recommended relationships
- missing traceability

### Step 6 – Review Governance Fit

Validate alignment with:

- applicable KTS standards
- CNS constraints
- existing governance decisions

### Step 7 – Make Recommendation

Select one outcome:

- Create
- Consolidate
- Convert to another object type
- Do Not Create

## Success Criteria

Knowledge objects are:

- non-duplicative
- well-related
- appropriately classified
- governance-aligned
- AI-consumable
